# JW_cad 連携・効率化の調査と提案

## 結論(先に要点)

- JW_cad には Web API のような外部プログラム連携の公式 API は**ありません**。連携手段は実質2つです。
  1. **座標ファイル読み込み**(テキストを読み込んで点を配置)← 本アプリの txt がそのまま使えます。
  2. **外部変形(がいぶへんけい)**(`JWC_TEMP.TXT` を介して bat / Python 等を呼び、作図を自動化)。
- 測量座標は JW_cad 上で **X(縦・北)と Y(横・東)が逆**になります。本アプリは「写真通り 左X・右Y」で出力するため、**JW側では「YX読取」で読み込む**か、後述の変換スクリプトで並べ替えます。

---

## 1. 座標ファイル読み込み(まず確実な方法)

本アプリが出力する txt(各行「X Y」)を JW_cad で読み込みます。

手順の目安:
1. JW_cad メニュー →「設定」→「座標ファイル」または作図系プラグインの座標読取機能を開く。
2. 単位を **m** に設定。
3. **「YX読取」**(Y→X の順で読む)を選んで本アプリの txt を指定。
   - これで測量座標(X=北)を JW の画面座標(横=X)に正しく変換して配置できます。
4. 点番号・点名を振るかを選んで実行。

> 補足: 「XY読取」で読むと縦横が入れ替わって作図されます。必ず **YX** を選ぶか、`tools/to_jw.py` で並べ替えた版を使ってください。

---

## 2. 外部変形による自動作図(まとめて効率化する提案)

### 仕組み
JW_cad の「その他 → 外部変形」で bat を指定すると、JW は選択情報を `JWC_TEMP.TXT` に書き出して bat を呼び、bat(が呼ぶスクリプト)が同ファイルに作図データを書き戻すと、その内容が図面に反映されます。Perl / Python / VBScript などで記述できます。

### 提案する自動化
本アプリの座標 txt を入力に、ワンクリックで以下をまとめて実行する外部変形を用意できます。

- 全点を**点マーカー＋点番号**として一括配置
- 隣接点を順に結んで**外周ポリライン**を作図(敷地・区画の閉合)
- 閉合図形から**座標求積(面積計算)**を実行し、面積を図面に記入
- レイヤ分け(点/番号/外周を別レイヤ)

これにより「テキスト読み込み → 点打ち → 線引き → 求積」を手作業でやる工程が 1 アクションになります。

### 実装メモ
- `tools/to_jw.py` に、本アプリ txt を JW 直接読取用(YX並べ替え済み)へ変換する処理を用意しました。
- 外部変形の出力コマンド(点・線・文字の記法)は JW_cad のバージョンで細部が異なるため、実装時に対象環境の外部変形仕様に合わせて確定します。まずは「座標ファイル読み込み」で運用を固め、頻度が高い工程から外部変形へ寄せるのが安全です。

## 3. Claude API を使ったさらなる効率化(提案)

- **複数ページ一括処理**: 写真/PDF を連続投入し、ページ順に結合した 1 本の txt を自動生成。
- **表・範囲の自動判別**: 「B.1〜B.36 の表だけ」「赤線が引かれた行だけ」等の指示を自然言語で受け取り、対象行のみ抽出。
- **読取の自己検証**: 抽出後にもう一度モデルへ渡し、桁数・符号・小数点の整合、点数の一致をクロスチェックして信頼度を補正。
- **外部変形コマンドの自動生成**: 座標から点・線・文字の作図コマンド列を生成し、外部変形にそのまま渡す。
- **面積の突合**: 求積表(座標求積表)の画像から面積も読み取り、座標から計算した面積と一致するか検算。
